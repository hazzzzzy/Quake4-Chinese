Quake 4 汉化第九轮（承接 [[quake4-r6-crash-and-subtitles]] v1.0.5）：字体基线对齐 v1.0.6 + 仓库开源发布 + Strogg 士兵机制确认。

**Why:** 度量陷阱（用 M top 判 ASCII/CJK 谁偏上、方向反了）容易复发；仓库剥离逻辑与 postinstall 补齐流程建立标准；Strogg 士兵"改造后能听懂"是坊间误解，实际是 PA/无线电。

**How to apply:**

- **MCC 着陆场 混排 MCC 顶部偏上（v1.0.6 定案）**：loading 地名走 `guis/loading/generic.gui` 的 `t_levelname`（rect 290,11,336,20 / textscale 0.36 / font fonts/marine → **marine_24** 走 medium 档，不是 48）。**度量陷阱**：直接比 M top（36）vs CJK top（42）会得出"CJK 更高"结论——但用户观察 "MCC 偏上" 指的是**相对 rect 中线**而非相对 CJK。正推：rect(y=11,h=20)/textscale 0.36 → useScale=0.72，基线 y_bl = 21×0.72+11 = 26.12；M 位图矩形 = y_bl-18×0.72 到 y_bl+0 → 中线 = rect+8.64；着字（drop=2）位图矩形 = y_bl-21×0.72 到 y_bl+5×0.72 → 墨迹中线 ≈ rect+10.08 ≈ **rect 中线 10（居中）**；差 1.44 虚拟 = 3.24 屏幕 px（1080p 纵向缩放 2.25）。**正解**：`export_font.py` 全档 `ascii_drop = drop`（12→1、24→2、48→4 名义 px），ASCII 基础段位图顶烘同幅透明行、top 度量不变。实机 shot00336 验证：Sanchez 列兵 / Raven 小队 / 传入 通讯 混排完全齐平；HUD 数字 8/14/100/50（chain_24 drop=2 → h+=2）rect 仍兜住无裁切。教训（**不能重复的错误**）：不要用「字符 top 差」判断视觉位置——rect 中线才是用户观察基准；用 M 参考的智能算法（top_m - top_ref + drop）会得出 ascii_drop=0（因 M top < CJK top）→ 完全没修。r6 memory 里说的"仅对 CJK 用 drop"是当时 CJK 显著偏上（相对 rect）的临时定案，r9 补上对称的 ASCII 侧。
- **hazzzzzy/Quake4-Chinese public 仓库上线（2026-07-18）**：https://github.com/hazzzzzy/Quake4-Chinese，103MB / 75 files 首提 `7b37e58` + v1.0.6 `66d15c3`。仓库结构：`dist/`（可运行汉化根，剥离 5 类版权敏感物）+`src/`（translations tsv、tools py、engine-patches diii4a diff+Subtitles.h/cpp）+`docs/`（glossary + localization-guide 汉化制作方法论）+`screenshots/`。**剥离 5 类**（GitHub public 分发 DMCA 风险规避）：`q4game.dll.official`（Raven 原版备份）、`fonts/chinese/strogg_*.{fontdat,tga}`（原版外星文字体直通）、`guis/hud.gui`（基于原版 pak021 全文改）、`guis/maps/medlabs/med1_textchange.gui`（原版 pak001 全文改）、`zzz_vo_chinese_alias.pk4`（原版英文原声路径别名 65MB）。**补齐方案** `dist/postinstall.cmd` → `src/tools/build_dist_extras.py`：检测 python → 从玩家自己合法拥有的 pak001/pak021 现场生成到 savedata（strogg 字体直接提取、hud.gui/medlab.gui 提取后 apply 内嵌 patch 表、vo pk4 用 `ZipInfo` 重命名保留原压缩数据不解压重压 → 秒级完成）。玩家原版目录只读、汉化 100% 隔离。**关键**：hud.gui 补丁底稿必须用 pak021（1.4 版 2507 行），pak001 版（2272 行）结构差 235 行必崩存档（r4 教训）——patch 表已固化 2 处纯数值替换。**LICENSE 组合**：主体 GPL-3.0（引擎/DLL 感染），翻译文本 CC BY-NC-SA 4.0，CJK 字体位图 SIL OFL 1.1（派生自思源黑体 Medium OTF）；LICENSE-FONTS.txt 独立说明。**引擎侧改动**：6 M（CMakeLists.txt/Game_local.cpp/LipSync.cpp/Misc.cpp/Sound.cpp/ai/AI.cpp）+ 2 新增（Subtitles.h/cpp）在 diii4a `v1.1.0harmattan70` tag 之上，作为 `0001-quake4-cn-runtime.patch` + 新文件全文分发（GPL 必需）。**gh 命令**：本地 git identity `git config user.name/user.email`（NEVER `--global`），gh 已登录 hazzzzzy 有 repo scope；`gh repo create --public --source=. --push` 一步完成。**gitignore 陷阱**：`*.tga` 全局屏蔽会误杀可发布字体 tga，必须精确到 `tests/**/*.tga` / `dist/savedata/**/screenshots/*.tga`（不然 gitattributes 里没啥办法救）。**PowerShell 5.1 中文 heredoc 坑**：`@'...'@` 里含中文的 commit message 传给 git 会被拆成 pathspec 报错；改用 `git commit -F <file>` + UTF-8 文件。
- **Strogg 士兵 speech 机制（用户误解澄清）**：monster_strogg_marine.def 里全是 `snd_grunt/gear/pain/breathe/weapon_fire`，**无 speech/lipsync 挂点**；`sound/strogg_marine.sndshd` 里的 sound shader（smarine_alert/pain/die）description 是**开发者调侃注释**（"i must kill now"/"ouchie"/"sometimes i fall down"），**音频文件是 sound/strogg/stroggmarine/sight3.ogg 等外星语哼叫声**（非英文台词）。所以"Kane 改造后能听懂 Strogg 士兵近距离对话"是**坊间误解 + 官方设定说法**——原版实际没有对应资产。用户听得到的"清晰 Strogg 对话"是 PA 广播和 Strogg 无线电（如 medlabs 的 neurocyte implanted、Nexus 播报），这些**已汉化**：zz_chinese_radio.lipsync 251 条 + zz_chinese_aivo.lipsync 668 条 + dialogue_lips.tsv 3962 条全量翻译。**结论**：无需补 Strogg 士兵专项汉化；用户若在特定关卡看到无字幕的 Strogg 说话截图，再单点排查是 aiSpeak decl 缺失还是 speaker 未挂钩。
- **实机 cfg 帧驱动坑（再补充）**：(1) `+exec CFG.cfg` 命令行参数在 PowerShell Start-Process ArgumentList 里传含空格路径的 `+set fs_basepath "D:\data\Quake 4"` 时，args 被拆导致 exec 参数被吃掉；解法=**改用 .cmd 批处理**做 launcher（run_apx_chinese.cmd 是模板），CMD 解析空格路径可靠。(2) map 命令必须带 `game/` 前缀：`devmap game/mcc_landing` 而不是 `devmap mcc_landing`（原版 log "Can't find map maps/mcc_landing.map"）。(3) loading 屏期间 `wait N` 引擎主线程 busy 加载、服务器帧不推进，screenshot 挂到加载完才执行——**拿不到 loading 屏本身**；折中方案 `testGui guis/loading/generic.gui` 关卡内叠加显示（但 `gui::loading_levelname` 需 loading 屏正式路径才设值，testGui 时 t_levelname 为空）。所以 loading 地名对齐**只能通过间接验证**：同关卡内 marine_24 混排（准星名牌 Sanchez 列兵、无线电 传入 通讯）齐平即证明 loading 地名齐平（同一份 fontdat）。
